package com.youkeda;

import java.util.HashMap;

public class Demo {

public static HashMap<Integer, Integer> cache = new HashMap<>();

// 一夜之间获取的最高现金
public static int f(int[] rooms) {
    // 每次调用的时候，将cache情况
    cache.clear();
    // 从右侧开始考虑是否偷盗，有新的思路也可以按照自己思路来
    return fCore(rooms, rooms.length - 1);
}

public static int fCore(int[] rooms, int index) {
  int i;
  i = rooms.length;
  if(i == 0){
    return 0;
  }
  if(i == 1){
    return 1;
  }
  if(i == 2){
    return Math.max(rooms[0] , rooms[1]);
  }
 else {
   int[] money = new int[i];
   int n;
   for(n = 0; n < i; n++){
     money[n] = 0;
   }
   money[0] = rooms[0];
   money[1] = Math.max(rooms[0],rooms[1]); 
   for (n = 2;n < i ; n++ ) {
     money[n] = Math.max(rooms[n] + money[n - 2], money[n - 1]);
   }
   return money[i-1];
 }


}

public static void main(String[] args) {
    int[] rooms1 = {1, 2, 3, 1};
    System.out.println("{1,2,3,1}" + f(rooms1));
    int[] rooms2 = {2, 7, 9, 3, 1};
    System.out.println("{2,7,9,3,1}" + f(rooms2));

}

}
本次的算法编程是在前一天课程的基础上实现的，本次的算法题目对于我来说其实只是有一个大概的思路，通过百度到这个案例进行理解才完成的今日的题目。
思路：相邻两间房不能同时偷，所以在数组上有一个i和i-2的一个关系，第一个偷不偷就取决于i和i-2的和与i-1的大小比值关系。如果i+i-2>i-1则第一个房子偷，反之从第二个房子开始。
总的来说就是一个数组不相邻的集合的最大值。
遇到的问题：一开始看到百度的算法觉得我还是很聪明的，思路差不多，但是后来自己编程的时候就有不少的问题，对于概念的模糊，数组的定义问题，虽然表述没有错，但是在实现功能的时候出现了一些问题，好几次的超出下标的界限，int与int[]传值的问题，以及在最后的金额相加的部分，有money[]和rooms[]，因为money初值是0，需要加入房屋自身的价值，再进行比较，所以在最后算价值的地方需要注意这个问题。
